# 実装前ドキュメント更新

実装タスクに着手する **前に** 、関連ドキュメントを確認・更新する。
ドキュメントが最新でない状態でコードを書き始めると、命名や設計がブレる原因になる。

## 実行タイミング

- 新機能の実装を始める前
- 新しいドメイン概念やテーブルを追加する前
- ユーザーから「実装して」と言われたら、コードを書く前にまずこのスキルを実行する

## 手順

### 1. ユビキタス言語の確認・更新

`.claude/rules/ubiquitous-language.md` を読み、これから実装する機能に関わる用語がすべて定義されているか確認する。

- 新しい概念があれば **用語・英語名・説明・例** を追加する
- 既存の用語に変更があれば更新する
- コード上の英語名 (変数名・テーブル名) はここで確定させる

### 2. ドメイン知識の確認・更新

`docs/domain.md` を読み、実装する機能に関連するドメイン知識が記載されているか確認する。

- 不足があれば追記する
- 誤りがあれば修正する

### 3. 要件定義の確認

`docs/requirements.md` を読み、実装する機能の要件が明確に定義されているか確認する。

- 要件が曖昧な場合はユーザーに確認してから更新する
- データモデルに新しいテーブル・カラムが必要な場合、ここに追記する

### 4. 更新のコミット

ドキュメントに変更があった場合、実装コミットとは **別に** ドキュメント更新としてコミットする。

## 出力

更新内容をユーザーに簡潔に報告する:

- 追加・変更したユビキタス言語の一覧
- domain.md / requirements.md の変更箇所
- 「ドキュメント更新完了。実装に入ります。」で締める
