# ユビキタス言語 (Ubiquitous Language)

コード・コミットメッセージ・ドキュメント・会話すべてで以下の用語と英語名を統一して使うこと。

## ドメイン用語 → コード上の英語名

| 日本語 | 英語名 (コード) | 説明 | 例 |
|--------|----------------|------|-----|
| リリース | `release` | シングルまたはアルバム。ミーグリの起点となる CD 作品 | 17th シングル「クリフハンガー」 |
| リリース種別 | `releaseType` | `single` or `album` | `single` |
| イベント | `event` | 1 リリースに紐づくミーグリの開催単位。オンライン/オフライン別に分かれる | 17th オンラインミーグリ |
| 開催形式 | `eventFormat` | `online` or `offline` | `online` |
| 開催日 | `eventDay` | イベント内の各日程 (DAY1, DAY2, ...) | DAY3 = 2025/6/14 |
| 部 | `slot` | 1 開催日内の時間帯スロット (第①部〜第⑥部) | 第③部 (13:45〜15:00) |
| 受付次 | `receptionRound` | チケット申込の回 (第1次, 第2次, ...) | 第2次受付 (5/6 14:00〜5/7 14:00) |
| 最終受付 | `finalRound` | ある開催日に応募できる最後の受付次 | DAY1 の最終受付 = 第8次 |
| メンバー | `member` | 日向坂46 のメンバー | 坂井新奈 |
| 期 | `generation` | メンバーの加入期 (一期生, 二期生, 三期生, 四期生) | 四期生 |
| 推しメン | `oshiMember` | ユーザーが応援しているメンバー | ユーザー A の推しメン = 坂井新奈 |
| 参加記録 | `participationRecord` | ユーザーが記録するミーグリの申込・参加結果 | DAY3 第③部 坂井新奈 → 当選 |
| 当選 | `won` | 抽選に当選した状態 | — |
| 落選 | `lost` | 抽選に落選した状態 | — |
| 申込済 | `applied` | 受付次に申し込んだ状態 (当落結果待ち) | 第2次で DAY3 第③部に申込済 |
| 未申込 | `notApplied` | まだ申し込んでいない状態 | — |
| キャンセル | `cancelled` | 当選後にキャンセルした状態 | — |
| スキーマバージョン | `schemaVersion` | DB スキーマの互換性管理用バージョン | v2 (テーブル追加時にインクリメント) |
| 完売 | `soldOut` | ある枠の全チケットが埋まった状態 | DAY3 第③部 坂井新奈 = 完売 |
| 全完 | `allSoldOut` | あるメンバーの全開催日・全部が完売した状態 | 坂井新奈 17th 全完 (36/36) |
| 投げる | — | チケットを申し込むこと (UI 上の表現としてのみ使用、コードでは `apply`) | 「DAY3 に 3 枚投げる」 |
| 購買計画 | `purchasePlan` | 理想・確保・受付次別のマトリックスを含む計画 | 17th 購買計画 |
| 理想マトリックス | `idealMatrix` | 目標部数を開催日×部で配置したもの | DAY3×第③部 = 目標 1 枚 |
| 確保状況マトリックス | `actualMatrix` | 確保済み部数を開催日×部で配置したもの | DAY3×第③部 = 確保 0 枚 |
| 受付次別計画 | `roundPlan` | ある受付次で何部投げるかの計画 | 第2次: DAY3×第③部 に 2 枚投げる |
| 予算 | `budget` | 合計費用の試算 | 12 枚 × 1,200 円 + 送料 700 円 = 15,100 円 |

## 使い分けルール

- **コード (変数名・型名・テーブル名)**: 英語名の `snake_case` (DB) または `camelCase` (TypeScript) を使う
- **UI テキスト**: 日本語を使う。「投げる」等のファン用語は UI で自然なら使ってよい
- **コミットメッセージ・PR**: 日本語で書く場合は上記日本語をそのまま使い、表記ブレしない
- **新しい概念が出たらこのファイルに追加してから実装に入る**
